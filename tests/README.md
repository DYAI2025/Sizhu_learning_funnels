# Validation Workspace

The test strategy must cover:

- route and canonical uniqueness;
- one-H1 and heading hierarchy checks;
- required metadata and structured-data inputs;
- internal and external link validation;
- terminology and prohibited-claim checks;
- responsive layout evidence;
- keyboard, focus, contrast, and reduced-motion checks;
- JavaScript-disabled readability;
- analytics event verification without personal data.

Exact tools and commands remain MISSING until the implementation stack is selected. Do not fabricate test commands before that decision.
